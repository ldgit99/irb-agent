# ChatGPT Content Agent

## 역할
- IRB 계획서 본문을 ChatGPT(OpenAI API)로 작성한다.
- 출력은 반드시 JSON `sections` 객체로 반환한다.

## 필수 규칙
1. `OPENAI_API_KEY`를 사용해 OpenAI Responses API를 호출한다.
2. 응답은 마크다운이 아닌 JSON 객체만 허용한다.
3. 아래 11개 섹션 키를 정확히 사용한다.
4. 템플릿의 작성 지침을 문장에 직접 반영한다.
5. 템플릿의 설명/예시 문구를 그대로 복사하지 말고, 연구 주제에 맞춰 구체화한다.

## 섹션 키
- `1. 연구 과제명(국문)`
- `2. 연구 배경 및 목적`
- `3. 연구 설계 및 방법`
- `4. 연구대상자 선정 및 제외 기준`
- `5. 모집 및 동의 절차`
- `6. 예측 가능한 위험 및 불편과 최소화 대책`
- `7. 개인정보 및 연구자료 보호 계획`
- `8. 자료 보관 및 폐기 계획`
- `9. 보상 및 배상 계획`
- `10. 연구 수행 기관 및 소속`
- `11. 검토 메모`

## 템플릿 기반 상세 작성 지침
- 공통:
  - 연구 목적, 가설(있다면), 정당성을 구체적으로 기술한다.
  - 선행연구 또는 실무 근거를 반영해 문장의 타당성을 높인다.
  - 해당 없는 항목은 `해당없음`으로 명시한다.
- `2. 연구 배경 및 목적`:
  - 왜 필요한 연구인지, 기존 연구의 한계와 본 연구의 기여를 분리해 작성한다.
  - 최소 10문장 이상으로 작성한다.
  - 다음 사항을 반드시 반영한다.
    - 연구 목적을 구체적으로 기술
    - 의도하는 가설이 있다면 설명
    - 가설을 입증하기 위한 설명을 구체적으로 작성
    - 다른 연구의 선행 연구로서의 성격(해당 시) 기재
- `3. 연구 설계 및 방법`:
  - 연구 설계, 대상, 자료수집 방법, 도구, 절차, 분석 방법을 포함한다.
  - 최소 30문장 이상으로 작성한다.
  - 모든 시술/처치/행위에 관한 구체적 사항(연구대상자가 해야 할 일, 소요시간)을 포함한다.
  - 윤리적 고려사항과 연구자의 관점, 대응 방안을 포함한다.
  - 연구 수행 장소와 안전한 수행 절차를 포함한다.
  - 아래 순서를 유지해 작성한다.
    - 1) 연구 설계 및 연구 대상
    - 2) 자료수집 방법
    - 3) 연구도구
    - 4) 연구 계획 및 절차
    - 5) 자료분석 방법
  - 연구대상자 수와 산출 근거(선행연구/통계/질적연구 최소 필요 인원)를 제시한다.
  - 비교군/무작위/눈가림/시험물질 항목은 해당 시 구체화, 미해당 시 `해당없음` 처리한다.
- `4. 연구대상자 선정 및 제외 기준`:
  - 선정기준과 제외기준을 분리해 쓰고, 취약대상자 포함 시 정당성을 명시한다.
- `5. 모집 및 동의 절차`:
  - 모집 경로(기관 승인, 온/오프라인 게시 등)와 동의 획득 단계(설명, 질의응답, 동의서 확보)를 작성한다.
  - 동의 철회권과 불이익 없음 원칙을 반드시 포함한다.
  - 민감정보 수집 시 별도 동의 필요성을 명시한다.
- `6. 예측 가능한 위험 및 불편과 최소화 대책`:
  - 신체적/심리적/사회적/법적 위험을 포함해 작성한다.
  - 개인정보 유출 위험을 위험 항목으로 포함한다.
  - 대면 연구면 감염병(예: 코로나19) 안전대책을 포함한다.
- `7. 개인정보 및 연구자료 보호 계획`:
  - 최소수집, 가명처리, 접근권한, 암호화, 보관위치, 접근 로그 등 관리 방안을 구체적으로 작성한다.
- `8. 자료 보관 및 폐기 계획`:
  - 보관기간, 책임자, 전자파일 안전삭제, 종이문서 파쇄 등 폐기 방법을 분명히 쓴다.
  - 인체유래물 연구면 인체유래물 보관/폐기 계획을 포함한다.
- `9. 보상 및 배상 계획`:
  - 보상(참여에 대한 지급)과 배상(손상 발생 시 조치)을 구분해 작성한다.

## 품질 기준
- 각 섹션은 최소 3문장 이상, 실무적으로 바로 검토 가능한 수준의 구체성을 가진다.
- 모호한 표현(예: 적절히, 충분히)만 단독으로 쓰지 말고 실행 방법을 함께 쓴다.
- 수치/기간/절차/책임 주체 중 최소 2개 이상을 각 핵심 섹션에 포함한다.

## 입력
```json
{
  "topic": "string",
  "method": "string",
  "institution": "string",
  "department": "string",
  "sample_size": 100,
  "duration_months": 6,
  "template_guidelines": ["string"],
  "template_examples": ["string"]
}
```

## 출력
```json
{
  "sections": {
    "1. 연구 과제명(국문)": "string",
    "2. 연구 배경 및 목적": "string",
    "3. 연구 설계 및 방법": "string",
    "4. 연구대상자 선정 및 제외 기준": "string",
    "5. 모집 및 동의 절차": "string",
    "6. 예측 가능한 위험 및 불편과 최소화 대책": "string",
    "7. 개인정보 및 연구자료 보호 계획": "string",
    "8. 자료 보관 및 폐기 계획": "string",
    "9. 보상 및 배상 계획": "string",
    "10. 연구 수행 기관 및 소속": "string",
    "11. 검토 메모": "string"
  }
}
```

## 실패 처리
- JSON 파싱 실패: 코드블록 제거 후 재파싱 1회.
- 키 누락: 누락 키만 재요청하거나 전체 재생성.
- 빈 문자열: 해당 섹션 재생성.
